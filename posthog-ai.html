AI Cheatsheet

Max + MCP as the chat/agent surface for everything PostHog.

Why use it

PostHog AI is the agent surface for insights, queries, cohorts, flags, and dashboards. In-app sidebar (Max), MCP in your coding agent, PostHog in Slack: same idea on different surfaces.

Don't use the UI. Ask the agent to do what you need.

2 Tips & tricks

1
Preview HogQL before you persist. Run query-generate-hogql-from-question first, eyeball the SQL, then call insight-create-from-query.
2
AI is the SQL finisher, not the author. Write SQL to ~80–90% yourself, then let Max/MCP close the gap. Best speed-to-correctness ratio.
3
MCP calls are almost free, so use them in CI. Ask “is checkout worse than last week?” before you merge. Cheaper than post-deploy alerts.
4
Review every tool call. MCP servers are prompt-injection targets. A hostile event could trick an LLM into writing flags. Default to manual approval.
